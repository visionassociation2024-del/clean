# EliteClean

منصة ثنائية اللغة لحجز وإدارة الخدمات الميدانية، مبنية بـ HTML وTailwind CSS وJavaScript وExpress وPostgreSQL.

## التشغيل المحلي

1. انسخ `.env.example` إلى `.env` واضبط القيم. لا تُعد استخدام رابط قاعدة البيانات الذي كان مكشوفاً في النسخة السابقة؛ أنشئ كلمة مرور Neon جديدة أولاً.
2. حمّل متغيرات البيئة في الطرفية أو من خلال منصة التشغيل.
3. شغّل:

```powershell
npm install
npm start
```

ثم افتح `http://localhost:4000`. يجب فتح ملفات الواجهة من الخادم، وليس مباشرةً عبر `file://`، لأن الواجهة تستخدم مسارات API من نفس النطاق.

## متغيرات الإنتاج المطلوبة

- `DATABASE_URL`
- `ADMIN_USERNAME`
- `ADMIN_EMAIL`
- `ADMIN_PASSWORD`
- `ADMIN_TOKEN_SECRET` (قيمة عشوائية طويلة)
- `ALLOWED_ORIGINS` (النطاقات المسموحة مفصولة بفواصل)
- `NODE_ENV=production`
- `AUTO_INIT_DB=false` بعد تنفيذ `backend/schema.sql` مرة واحدة على قاعدة الإنتاج

## الفحص

```powershell
npm run check
npm test
```

نقطة فحص الخدمة: `GET /api/v1/health`.

## النشر على Vercel

أضف متغيرات البيئة السابقة إلى إعدادات المشروع ثم انشر باستخدام `vercel.json`. ملف الخادم يصدّر تطبيق Express للاستخدام كدالة Serverless، ويبدأ منفذاً محلياً فقط عند تشغيله مباشرة.

## ملاحظات أمنية

- بيانات دخول الإدارة ليست مخزنة في الواجهة أو قاعدة البيانات؛ مصدرها متغيرات البيئة.
- جلسة الإدارة مدتها 8 ساعات وتُحفظ في `sessionStorage`.
- جميع مسارات `/api/v1/admin/*` محمية بتوقيع HMAC.
- بوابات الدفع وSMS والبريد الفعلي تحتاج حسابات ومفاتيح مزود خارجي قبل تفعيلها. السجلات الحالية تُحفظ في جدول `notifications` كطابور جاهز للربط.
